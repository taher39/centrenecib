
# مركز نسيب — منصة الحجز و الإدارة

تطبيق ويب احترافي (PWA قابل للتثبيت) بثلاث لغات (عربي/فرنسي/إنجليزي) مع زر تبديل في الأعلى، تصميم عصري بالأخضر الطبي + الأبيض الملكي، ولمسات ذهبية للعروض. يعتمد على **Lovable Cloud** للبيانات والمصادقة والصلاحيات وسجل النشاطات.

---

## 1. الهوية البصرية و التصميم

- **الألوان**: أخضر طبي (مستوحى من اللوغو) + أبيض ملكي + ذهبي للعروض + أحمر للديون/الأسعار المخفضة + أصفر للطلبات قيد الانتظار.
- **اللوغو**: يُحفظ في `src/assets/logo.png` ويظهر دائري على اليسار في كل الواجهات.
- **الخطوط**: عربي أنيق (Tajawal/Cairo) + لاتيني (Cormorant + Inter) لإحساس راقٍ طبي.
- **انيميشن مخصص**:
  - **شاشة دخول الزبون**: وردة زهرية تتفتح في المنتصف ثم تنشطر الشاشة لنصفين (يسار/يمين) لتكشف نموذج الدخول.
  - **نموذج الطلب**: نصفان يأتيان من اليسار واليمين ويلتقيان في المنتصف.
  - **تأكيد الطلب**: الأزرار تنسحب يساراً ويميناً ثم تظهر رسالة النجاح.
  - **شريط صور** متحرك أعلى صفحة الزبون (10 صور يحددها الأدمن).
  - **بانر العروض**: يظهر 5 ثوانٍ ويختفي 6 ثوانٍ بشكل دوري، بإطار ذهبي وخط أسود وسعر أحمر، مع عد تنازلي.

---

## 2. تجربة الزبون

### تسجيل الدخول (`/`)
- **زبون جديد**: اسم كامل + عمر + رقم هاتف → عند نجاح أول حجز يُمنح **كود مكوّن من 10 أرقام** يُحفظ تلقائياً في المتصفح (localStorage) + يُعرض في رسالة التأكيد.
- **زبون عائد**: إدخال الكود مباشرة (أو يُملأ تلقائياً) → دخول مباشر.

### اختيار الجلسة و الحجز
- يرى الجلسات المتاحة (نوع، مدة، سعر بالدج، اليوم/الأيام التي حددها الأدمن).
- يختار **جلسة أو أكثر** (متعددة).
- خوارزمية الحجز الذكي:
  - الأدمن يضبط ساعات العمل (مثلاً 9:00–12:00 و 14:00–17:00) و **سعة كل جلسة** (مثلاً 3 أشخاص في النفس الموعد).
  - أول 3 يُوضعون في 9:00، الزبون رقم 4 يُوضع تلقائياً في 9:00 + مدة الجلسة (مثلاً 9:45)، وهكذا.
- رسالة التأكيد تعرض كل المواعيد: "التنظيف العميق: السبت 14:30 — التدليك: الإثنين 10:00…" + الكود.

### صفحة الزبون الشخصية (بعد تثبيت التطبيق أو الدخول بالكود)
- شريط صور متحرك أعلى الصفحة.
- بانر العروض الذهبي مع العد التنازلي.
- جلساته القادمة + السابقة + الديون المتراكمة إن وُجدت.
- زر **"تثبيت التطبيق"** (PWA install prompt) يختفي بعد التثبيت.

## 3. لوحة الأدمن

### مصادقة الأدمن
- اسم مستخدم + كلمة مرور (قابلة للتعديل لاحقاً).
- صفحة منفصلة `/admin/login`.

### المواعيد (`/admin/appointments`)
- تبويبات: **اليوم — الغد — هذا الأسبوع — الكل — الملغاة — المؤجلة**.
- شارة إشعار حمراء فوق "المواعيد" بعدد الطلبات الجديدة (قيد الانتظار) حتى تُقرأ.
- لكل موعد حالة قابلة للتغيير: **قيد الانتظار / مؤكد / منجزة / ملغى / مؤجل**.
  - **منجزة** → توليد فاتورة فرنسية تلقائياً.
  - **ملغى** → الموعد الذي يليه يصعد ليأخذ المكان (إعادة ترتيب طابور).
  - **مؤجل** → اختيار يوم/وقت جديد يدوياً أو تلقائياً في اليوم المخصص للجلسة.

### الجلسات و جدول العمل (`/admin/services`)
- إضافة/تعديل/حذف الجلسات (الاسم، المدة، السعر، السعة، الأيام المتاحة).
- ضبط ساعات العمل العامة.

### العروض (`/admin/offers`)
- إضافة عرض: اسم، صورة، سعر مخفّض، مدة العرض (مع تاريخ انتهاء للعد التنازلي).

### الصور (`/admin/gallery`)
- رفع/حذف حتى 10 صور للشريط المتحرك في صفحة الزبون.

### العملاء (`/admin/clients`)
- ترتيب تلقائي:
  1. **أحمر** — لديهم ديون (الأعلى دائماً).
  2. **أصفر** — لديهم طلبات قيد الانتظار.
  3. عادي — البقية.
- بحث بالاسم أو رقم الهاتف.
- لكل عميل **سجل علاجي**: ملاحظات الأدمن + كل الجلسات السابقة و القادمة + الفواتير + الديون.

### الفواتير (`/admin/invoices`)
- فاتورة فرنسية أنيقة بمعايير جزائرية:
  - رأسية: اسم المركز + اللوغو + العنوان + الهاتف + **NIF / NIS / RC / Article / AI** (قابلة للتعديل من الإعدادات).
  - رقم فاتورة + تاريخ + بيانات الزبون.
  - الجلسات + السعر الإفرادي + المجموع + التخفيض إن وُجد + الصافي.
  - حالة الدفع: **كلي / جزئي** (مع المبلغ المدفوع و المتبقي = دين).
  - مكان للإمضاء أسفل الفاتورة.
  - تصدير PDF / طباعة.

### التخفيضات (`/admin/discounts`)
- جدول: اسم الزبون + رقم الفاتورة + المبلغ المخفّض + التاريخ.
- في الأعلى: مجموع التخفيضات الإجمالي.

### صافي الحساب (`/admin/finance`)
- **المداخيل**: تلقائية من الفواتير المدفوعة + إدخال يدوي (مبلغ + سبب + تاريخ/وقت).
- **المخاريج**: إدخال يدوي (مبلغ + سبب + تاريخ/وقت).
- الصافي = مداخيل − مخاريج.
- رسم بياني شهري.

### الإعدادات (`/admin/settings`)
- بيانات المركز + المعطيات الجزائرية للفاتورة.
- تغيير اسم المستخدم/كلمة المرور.

### السكرتيرات (`/admin/staff`)
- إنشاء حساب (اسم مستخدم + كلمة مرور).
- صلاحيات دقيقة بصناديق اختيار: **عرض / تعديل / حذف** لكل قسم (المواعيد، العملاء، الفواتير، الحسابات، العروض…).
- تعديل/حذف الحساب.

### سجل النشاطات (`/admin/activity`)
- كل عملية تقوم بها السكرتيرة (إضافة/تعديل/حذف/تغيير حالة) تُسجّل تلقائياً: الفاعل + النوع + الكائن + الوقت.

---

## 4. لوحة السكرتيرة

- نفس واجهة الأدمن لكن مع إخفاء/تعطيل الأقسام والإجراءات حسب الصلاحيات المُمنوحة لها.

---

## 5. الترجمة (FR / AR / EN)

- زر تبديل اللغة أعلى يمين كل الواجهات.
- نظام i18n (مفتاح→نص) بثلاث ملفات JSON.
- الواجهة تتحول تلقائياً RTL/LTR عند التبديل بين العربية والأخرى.
- **الفاتورة المطبوعة تبقى فرنسية دائماً** (طلب صريح).

---

## 6. PWA (تطبيق قابل للتثبيت)

- `manifest.json` + service worker.
- زر "تثبيت التطبيق" يظهر للزبون ويختفي تلقائياً بعد التثبيت.
- يعمل ويتذكر الكود حتى لو أُغلق التطبيق.

---

## التفاصيل التقنية

- **Backend**: Lovable Cloud (Postgres + Auth + Storage + RLS).
- **جداول رئيسية**:
  - `clients` (id, code 10 digits, full_name, age, phone, notes, debt_total).
  - `services` (name, duration_min, price_dzd, capacity, available_days).
  - `working_hours` (slots).
  - `appointments` (client_id, service_id, date, time, status, created_at).
  - `invoices` (number, client_id, items, total, discount, paid, payment_type).
  - `payments` / `expenses` (manual finance entries).
  - `discounts` (invoice_id, amount).
  - `offers` (title, image, price, ends_at).
  - `gallery_images`.
  - `staff_accounts` (username, password_hash) + `user_roles` (مع enum للصلاحيات الدقيقة).
  - `activity_log` (actor, action, entity, timestamp, diff).
- **RLS**: الأدمن يقرأ كل شيء؛ السكرتيرة محدودة بصلاحياتها عبر دالة `has_permission(user, scope, action)` (security definer لتفادي العودية).
- **مصادقة الزبون**: عبر الكود (10 أرقام) — جدول `clients` نفسه + جلسة محلية مشفّرة بدون Supabase Auth للزبائن (تبسيط التجربة).
- **مصادقة الموظفين**: Supabase Auth (email/password حيث الـ email يُولّد من اسم المستخدم).
- **خوارزمية تخصيص الوقت**: دالة في خادم TanStack تأخذ (service_id, date) → ترجع أول فتحة متاحة حسب السعة + ساعات العمل.
- **توليد PDF للفاتورة**: مكتبة `@react-pdf/renderer` بقالب فرنسي مطابق للمعايير.
- **i18n**: `i18next` + `react-i18next` + `LanguageDetector`.
- **PWA**: `vite-plugin-pwa`.

## خارطة التنفيذ (داخل دورة واحدة قدر الإمكان)

1. تفعيل Lovable Cloud + إنشاء كل الجداول و RLS و enums للصلاحيات.
2. إعداد i18n + ملفات الترجمة الثلاث + زر التبديل + RTL.
3. الهوية البصرية (ألوان، خطوط، اللوغو، tokens في `styles.css`).
4. واجهة الزبون: شاشة الدخول بالأنيميشن + الحجز + تأكيد + صفحة شخصية + شريط الصور + بانر العروض + PWA install.
5. منطق الحجز الذكي (الخوارزمية + التحقق من السعة).
6. لوحة الأدمن: المصادقة + المواعيد + الإشعارات + تغيير الحالات + إعادة ترتيب الطابور عند الإلغاء.
7. الجلسات + ساعات العمل + العروض + المعرض.
8. العملاء + الترتيب الملوّن + البحث + السجل العلاجي.
9. الفواتير الفرنسية + PDF + التخفيضات.
10. الحسابات (مداخيل/مخاريج/صافي).
11. السكرتيرات + الصلاحيات + سجل النشاطات.
12. اختبارات يدوية لكل تدفّق + ضبط بصري نهائي.

> ملاحظة: الحجم كبير جداً. سأبني الجزء الأول الكامل في هذه الدورة وقد أحتاج دورة أو دورتين إضافيتين لإكمال كل التفاصيل (خاصة الفاتورة PDF + الصلاحيات الدقيقة + الترجمة الكاملة لكل النصوص). سأخبرك في نهاية كل دورة بما تبقى.
