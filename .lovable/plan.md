# خطة التحسينات الشاملة

سأقوم بتنفيذ التحسينات التالية على دفعات. الخطة مقسمة إلى 5 مراحل.

## المرحلة 1 — قاعدة البيانات (Migration واحد)

إضافة الأعمدة والجداول التالية:

- **`clients.gender`** — `text` (`'male' | 'female'`) — لتسجيل جنس الزبون
- **`clients.age`** — `int` (إن لم يكن موجودًا)
- **`appointments.attendance`** — `text` (`'present' | 'absent' | null`) لتسجيل الحضور اليومي
- **جدول `products`** — `id, name, price, active, created_at, updated_at`
- **جدول `invoice_items` تعديل** — السماح بـ `product_id` اختياري (إضافة منتجات للفاتورة)
- **جدول `medical_reports`** — تقرير طبي لكل زبون يحتوي:
  - `id, client_id (FK), description, notes, diagnosis, recommendations, created_at, updated_at`
- مع RLS + GRANT كاملة

## المرحلة 2 — صفحة الزبون (`/me`)

1. **عرض المواعيد القادمة والسابقة** بشكل صحيح مع التاريخ والوقت والحالة الكاملة (مؤكد/منجز/مؤجل/ملغى/قيد الانتظار) — التحقق من `clientDashboard` server fn
2. **استبدال الشريط الحالي** (الذي يعرض صورتين) بـ **Carousel احترافي** يعرض صورة واحدة فقط، يتغير تلقائيًا كل 5 ثوانٍ، مع إمكانية التمرير اليدوي، نقاط مؤشر، انتقالات سلسة (نفس النمط في `/` و `/me`)

## المرحلة 3 — تحديثات الصفحة الرئيسية (`/`)

- استبدال **"للعلاج التجميلي و العناية بالبشرة"** بـ **"للعلاج التجميلي و العناية الجسدية"** في جميع الترجمات (ar/fr/en)
- تكبير حجم النص ليكون أوضح
- إزالة **"حصرياً للنساء"** من صفحة التسجيل/الترحيب
- إضافة **حقل الجنس** (ذكر/أنثى) في نموذج تسجيل الزبون الجديد + حقل العمر إن لم يكن
- استبدال شريط الصور في الصفحة الرئيسية بـ Carousel احترافي

## المرحلة 4 — لوحة الادمن

### 4.1 صفحة المواعيد (`admin.index.tsx`)
- إضافة تبويبات: **طلبات جديدة | مؤكدة | منجزة | اليوم | الكل**
- إضافة زر **"إنشاء موعد جديد"** بنموذج:
  - بحث عن زبون موجود (بالاسم/الهاتف/الكود) → اختياره
  - أو إدخال زبون جديد (اسم، هاتف، عمر، عنوان، جنس) → ينشئ تلقائيًا مع كود
  - اختيار خدمة واحدة أو أكثر
  - اختيار التاريخ والوقت
- زر سريع لتغيير الحالة إلى **"حاضر"** لكل موعد

### 4.2 صفحة جديدة: الحضور (`/admin/attendance`)
- عرض كل المواعيد التي حالتها `attendance = 'present'` لليوم الحالي
- مع اسم الزبون، الهاتف، الخدمة، الوقت
- فلتر بالتاريخ

### 4.3 صفحة جديدة: المنتجات (`/admin/products`)
- CRUD للمنتجات (اسم، سعر بالدينار، نشط)
- في صفحة الفواتير: زر **"إضافة منتج"** لإضافة منتجات لأي فاتورة قبل/عند الدفع

### 4.4 صفحة جديدة: التقارير الطبية (`/admin/medical-reports`)
- قائمة بجميع الزبائن الذين لديهم جلسات منجزة
- نقرة على زبون → تقرير احترافي يعرض:
  - **بطاقة معلومات**: الاسم، الهاتف، العنوان، العمر، الجنس، الكود
  - **سجل الجلسات** الكامل بالتاريخ والوقت والخدمة والحالة
  - **حقول تحرير**: التشخيص، الوصف، التوصيات، الملاحظات (يحفظها الادمن)
  - تصميم قابل للطباعة على نمط الفواتير

## المرحلة 5 — التنقل والقوائم

- إضافة الصفحات الجديدة (الحضور، المنتجات، التقارير الطبية) إلى قائمة الادمن الجانبية
- إضافة صلاحيات جديدة في نظام `usePerms` (`products`, `attendance`, `reports`)

---

## ملاحظات تقنية

- لن أُغيّر أي ملفات auto-generated (`types.ts`, `client.ts`, إلخ)
- سأستخدم `Promise.all` و lookups في server functions لتجنب مشاكل العلاقات
- Carousel جديد باستخدام `embla-carousel-react` المتوفر بالفعل في المشروع
- التقرير الطبي سيستخدم نفس نمط `InvoicePrintView` للطباعة

العمل كبير لكنه مترابط. سأبدأ بـ:

2. Server functions الجديدة
3. صفحات الادمن الجديدة
4. تحديث صفحة الزبون والصفحة الرئيسية

أخبرني إن أردت تعديل/حذف أي جزء، أو "وافق" للبدء.
